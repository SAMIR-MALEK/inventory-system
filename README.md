## ما يحتوي عليه النظام

✅ **إدارة الوسائل** — إضافة وتعديل وحذف وتصنيف الوسائل مع الكميات  
✅ **إدخال وإخراج الوسائل** — شراء، توزيع، تحويل، إرجاع، إتلاف  
✅ **الجرد** — جرد المكاتب والقاعات والمخزن الرئيسي مع سجل تاريخي  
✅ **المستخدمون** — إدارة المستخدمين وأدوارهم (مدير / مستخدم عادي)  
✅ **التقارير** — إحصائيات Dashboard وتقارير قابلة للطباعة  
✅ **PWA** — يعمل في المتصفح بدون إنترنت ويمكن تثبيته على سطح المكتب  
✅ **RTL** — واجهة عربية كاملة من اليمين إلى اليسار

## التشغيل المحلي

### 🗄️ الخطوة 1: إعداد قاعدة البيانات (Supabase)

1. اذهب إلى https://supabase.com وأنشئ حساباً مجانياً
2. اضغط **New project** واختر اسماً وكلمة مرور قوية
3. انتظر حتى تنتهي إنشاء المشروع (دقيقة أو دقيقتان)
4. اذهب إلى **Settings → Database**
5. ابحث عن **Connection string → URI** وانسخه
   - يبدو هكذا: `postgresql://postgres:[كلمة-المرور]@db.xxxxx.supabase.co:5432/postgres`
6. احتفظ بهذا الرابط، ستحتاجه لاحقاً

---

### ⚙️ الخطوة 2: إعداد Backend

افتح **Command Prompt** أو **Terminal** واكتب:

```bash
# الانتقال إلى مجلد Backend
cd inventory-system/backend

# تثبيت المكتبات
npm install

# إنشاء ملف الإعدادات
copy .env.example .env
```

افتح ملف `backend/.env` بأي محرر نصوص وعدّله:

```env
DATABASE_URL="الرابط-الذي-نسخته-من-Supabase"
JWT_SECRET="اكتب-هنا-اي-كلمة-سرية-طويلة-مثلا-abc123xyz456"
PORT=5000
FRONTEND_URL="http://localhost:5173"
```

ثم تابع في Terminal:

```bash
# إنشاء جداول قاعدة البيانات
npx prisma generate
npx prisma db push

# إضافة البيانات الأولية (مستخدم مدير + فئات + مواقع)
node src/utils/seed.js

# تشغيل الخادم
npm run dev
```

يجب أن ترى: `✅ الخادم يعمل على المنفذ 5000`

---

### 🖥️ الخطوة 3: إعداد Frontend

افتح **Terminal جديد** (لا تغلق الأول) واكتب:

```bash
# الانتقال إلى مجلد Frontend
cd inventory-system/frontend

# تثبيت المكتبات
npm install

# إنشاء ملف الإعدادات
copy .env.example .env

# تشغيل الواجهة
npm run dev
```

افتح المتصفح على: **http://localhost:5173**

## النشر على الإنترنت

### 🐙 الخطوة 1: رفع الكود على GitHub

1. اذهب إلى https://github.com وأنشئ حساباً
2. اضغط **New repository** واختر اسم مثل `inventory-system`
3. اختر **Public** ثم اضغط **Create repository**
4. اتبع التعليمات لرفع الكود:

```bash
cd inventory-system
git init
git add .
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/اسمك/inventory-system.git
git push -u origin main
```

---

### 🚂 الخطوة 2: نشر Backend على Railway

1. اذهب إلى https://railway.app وسجّل دخولاً بحساب GitHub
2. اضغط **New Project → Deploy from GitHub repo**
3. اختر مستودعك `inventory-system`
4. اضغط على المشروع واختر **Settings → Root Directory**: اكتب `backend`
5. اذهب إلى **Variables** وأضف هذه المتغيرات:

```
DATABASE_URL = [رابط Supabase]
JWT_SECRET = [كلمة سرية قوية]
PORT = 5000
FRONTEND_URL = https://[اسم-مشروعك].vercel.app
```

6. اضغط **Deploy** وانتظر حتى ينتهي
7. انسخ رابط Backend (يبدو مثل: `https://inventory-backend-xxx.up.railway.app`)

---

### ▲ الخطوة 3: نشر Frontend على Vercel

1. اذهب إلى https://vercel.com وسجّل دخولاً بحساب GitHub
2. اضغط **New Project** واختر مستودعك
3. في **Root Directory** اكتب: `frontend`
4. في **Environment Variables** أضف:

```
VITE_API_URL = https://[رابط-railway-الخاص-بك]/api
```

5. اضغط **Deploy** وانتظر دقيقتين
6. ستحصل على رابط مثل: `https://inventory-system-xxx.vercel.app`

7. **مهم**: ارجع إلى Railway وحدّث `FRONTEND_URL` برابط Vercel

## دليل الاستخدام

### 👤 إدارة المستخدمين
- من القائمة الجانبية اختر **المستخدمون** (للمدير فقط)
- يمكن إضافة مستخدمين جدد وتحديد دورهم (مدير أو مستخدم عادي)

### 📦 إدارة الوسائل
- من القائمة اختر **الوسائل والمخزون**
- اضغط **إضافة وسيلة** لإضافة وسيلة جديدة
- يمكن البحث والتصفية حسب الفئة والموقع والحالة

### 🔄 تسجيل العمليات
- من القائمة اختر **العمليات**
- اضغط **عملية جديدة** واختر نوع العملية:
  - **شراء**: إضافة كمية جديدة للمخزون
  - **توزيع**: إخراج وسيلة لمكتب أو قاعة
  - **تحويل**: نقل من موقع لآخر
  - **إرجاع**: إعادة وسيلة للمخزون
  - **إتلاف**: شطب وسيلة تالفة

### 🏷️ الفئات والمواقع
- أضف فئات الوسائل (أثاث، أجهزة إلكترونية، ...)
- أضف المواقع: مخازن، مكاتب، قاعات

### 📊 التقارير
- من القائمة اختر **التقارير**
- اضغط **طباعة التقرير** للحصول على نسخة ورقية

### 📱 تثبيت التطبيق
- افتح الموقع في المتصفح
- ستظهر رسالة "تثبيت التطبيق" - اضغط عليها
- سيُضاف التطبيق لسطح المكتب ويعمل بدون إنترنت

## 🆘 حل المشاكل الشائعة

**المشكلة**: `Cannot connect to database`  
**الحل**: تأكد من صحة رابط `DATABASE_URL` في ملف `.env`

**المشكلة**: `401 Unauthorized`  
**الحل**: تأكد من تشغيل `seed.js` وصحة بيانات الدخول

**المشكلة**: الواجهة لا تتصل بـ Backend  
**الحل**: تأكد من صحة `VITE_API_URL` في `.env` الخاص بـ frontend

**المشكلة**: خطأ عند رفع الكود على GitHub  
**الحل**: تأكد من تثبيت Git وإنشاء المستودع أولاً
